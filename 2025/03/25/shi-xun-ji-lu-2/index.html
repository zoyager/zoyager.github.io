Web安全学习日志 - 第二天

今日学习目标

  1. 掌握Web网站的核心组成部分
  2. 理解SQL注入漏洞的原理与分类
  3. 掌握SQL注入漏洞的探测与利用方法
  4. 熟悉MySQL数据库的基础操作

一、Web网站的构成

1. 基础架构

服务器 → 源码 → Web 容器 → 数据库

2. 核心组件

组件 说明
服务器 Windows Server/Linux服务器，承载网站运行环境
源码 - 前端：Vue/React/Webpack
- 后端：Spring/PHP/Node.js
数据库 MySQL/PostgreSQL/Redis，存储业务数据
Web容器 Apache/Nginx/Tomcat，处理HTTP请求与动态资源

3. 开源代码来源

  • GitHub：全球最大开源社区（如WordPress博客系统）
  • CMS平台：PHPCMS（门户）、SeaCMS（影视）、Hexo（静态博客）

二、MySQL基础操作

1. 数据库结构

库（Database） → 表（Table） → 字段（Column） → 数据（Data）

2. SQL语句基础

操作类型 示例语句 说明
查询 SELECT * FROM users WHERE id=1; 查询users表中id为1的数据
插入 INSERT INTO users (name) VALUES ('admin'); 插入新数据
删除 DELETE FROM users WHERE id=1; 删除指定数据
更新 UPDATE users SET name='root' WHERE id=1; 更新数据

3. 连接MySQL

  • 命令行：mysql -u root -p（输入密码）
  • 可视化工具：Navicat、SQLyog、phpMyAdmin
  • 集成环境：phpStudy、宝塔面板、XAMPP

三、SQL注入漏洞

1. 漏洞原理

  • 定义：用户输入的数据被拼接至SQL语句，导致恶意SQL执行。
  • 核心条件：
    1. 用户输入可控制（如URL参数id=1）
    2. 输入未被严格过滤（如单引号未转义）

2. 漏洞分类

类型 特点描述
联合注入 通过UNION SELECT获取数据，需页面回显支持
报错注入 利用SQL语法错误回显数据（如floor()函数）
盲注 无回显，通过布尔逻辑（AND/OR）或时间延迟（SLEEP()）判断结果
二次注入 注入数据存储后被二次使用（如写入数据库后触发）
堆叠注入 通过分号执行多条SQL语句（需数据库权限支持）
